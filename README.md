# NiriMod RU

**Русская версия GTK4/libadwaita-менеджера настроек для Wayland-композитора [niri](https://github.com/niri-wm/niri).**

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
[![Python 3.12+](https://img.shields.io/badge/Python-3.12%2B-blue?logo=python&logoColor=white)](https://python.org)
[![GTK4](https://img.shields.io/badge/GTK-4%20%2B%20libadwaita-4A90D9?logo=gnome&logoColor=white)](https://gtk.org)
[![Wayland](https://img.shields.io/badge/Wayland-native-orange)](https://wayland.freedesktop.org)

NiriMod RU помогает настраивать `~/.config/niri/config.kdl` через нативный графический интерфейс: мониторы, ввод, сочетания клавиш, расположение окон, рабочие пространства, автозапуск, переменные окружения, жесты, анимации и правила окон. Это не отдельная оболочка и не замена niri, а удобный редактор конфигурации, который сохраняет контроль за пользователем.

Проект является русскоязычной сборкой NiriMod. Цель форка - дать пользователям Arch Linux, CachyOS и других Linux-дистрибутивов тот же функционал, но с естественным русским интерфейсом, документацией и пакетной интеграцией.

![Главный экран NiriMod RU](docs/media/777shots_so.png)

## Возможности

- **Мониторы.** Настройка разрешения, частоты обновления, масштаба, позиции, трансформации, VRR и отключения отдельных выходов.
- **Ввод.** Параметры клавиатуры XKB, повтор клавиш, фокус при наведении, прокрутка, тачпад, мышь, курсор и автоскрытие курсора.
- **Сочетания клавиш.** Визуальная карта клавиатуры, список привязок, поиск, добавление и редактирование действий niri.
- **Внешний вид.** Тени, границы, скругления, прозрачность и эффекты размытия для окон.
- **Анимации.** Глобальное включение анимаций, замедление, редактор кривых easing, пресеты и настройка отдельных сценариев.
- **Расположение.** Промежутки между окнами, ширина колонок, центрирование, struts, фон и поведение CSD.
- **Рабочие пространства.** Именованные workspace и привязка рабочих пространств к мониторам.
- **Правила окон и слоёв.** Условия по App ID, заголовку или namespace, размеры, размещение, визуальные эффекты и поведение окон.
- **Автозапуск и окружение.** Команды запуска при старте niri и переменные окружения для процессов композитора.
- **Жесты и прочее.** Горячие углы, оверлей клавиш, путь сохранения скриншотов и настройки обзора.
- **Исходный конфиг.** Встроенный редактор KDL для ручных правок, когда быстрее написать настройку напрямую.

## Безопасная работа с конфигурацией

NiriMod RU рассчитан на аккуратное редактирование существующего `config.kdl`.

- Перед сохранением запускается `niri validate`; если проверка не проходит, изменения не применяются.
- Запись выполняется через временные файлы, чтобы снизить риск повреждения конфигурации.
- Нераспознанные блоки, пользовательские настройки и ручные правки сохраняются насколько позволяет текущий парсер.
- Поддерживаются резервные копии перед сохранением и восстановление из сохранённых копий.
- Есть отмена и повтор изменений внутри сессии.
- Если niri не запущен, изменения можно сохранить, но применить их нужно будет после запуска или перезагрузки конфигурации.

## Многофайловые конфигурации

NiriMod RU умеет читать конфигурации, разделённые через `include`, и старается сохранять изменённые параметры обратно в исходные файлы. Это полезно для пользователей, которые держат отдельные файлы для ввода, мониторов, правил окон, автозапуска или сторонних оболочек.

![Многофайловая конфигурация](media/multiple_configs.png)

Поддержка многофайловой конфигурации особенно важна для окружений с дополнительными слоями и пользовательскими shell-настройками, например Dank Material Shell или Noctalia. NiriMod RU редактирует только известные ему настройки niri и не должен специально переписывать ваши собственные блоки.

## Установка

### Arch Linux / CachyOS — сборка из PKGBUILD

В корне репозитория лежат готовые `PKGBUILD` и `.SRCINFO`. Ставится через `makepkg`, без обращения к AUR:

```bash
mkdir -p ~/build
cd ~/build
git clone https://github.com/y-tretyakov/nirimod-ru.git
cd nirimod-ru
makepkg -si
```

Что делает `-si`:

- `-s` (`--syncdeps`) — доставит недостающие зависимости через `pacman`, спросит sudo-пароль когда потребуется;
- `-i` (`--install`) — установит собранный пакет сразу после сборки, а не просто оставит `.pkg.tar.zst` в директории.

Полезные вариации:

```bash
makepkg -sirc   # + --clean: снести артефакты сборки после установки
makepkg -sricC  # + --cleanbuild: гарантированно чистая среда сборки с нуля
```

После установки приложение можно запустить из меню рабочего стола или командой:

```bash
nirimod
```

### Локальный пакет Arch (уже собранный `.pkg.tar.zst`)

Если пакет уже собран (например, на другой машине или в CI) и нужно просто установить готовый архив:

```bash
sudo pacman -U ./nirimod-ru-git-r91.20260703-1-x86_64.pkg.tar.zst
```

Имя файла может отличаться в зависимости от версии сборки.

### Другие дистрибутивы Linux

Для не-Arch систем используйте установочный скрипт:

```bash
curl -sSL https://raw.githubusercontent.com/y-tretyakov/nirimod-ru/main/install.sh | bash
```

Полезные флаги:

- `--install` - установка без интерактивного выбора действия.
- `--uninstall` - удаление установки, созданной скриптом.
- `--skip-deps` - не устанавливать системные зависимости автоматически.

## Требования

- Linux с Wayland.
- Установленный композитор `niri`.
- Python 3.12 или новее.
- GTK4, libadwaita, PyGObject и Pycairo.
- `uv` для установки через `install.sh` и запуска из исходников.
- `base-devel` (в частности `makepkg`) для сборки пакета из PKGBUILD.

Для Arch Linux зависимости подтягиваются пакетом при сборке (`makepkg -s`). Для других дистрибутивов `install.sh` пытается определить пакетный менеджер и установить недостающие зависимости через `apt`, `dnf`, `zypper`, `pacman` или `emerge`.

## Обновление

### Пересборка пакета (Arch Linux / CachyOS)

Пакет собирается из git-версии (`pkgver()` вычисляется через `git describe`), поэтому перед пересборкой стоит подтянуть свежий код:

```bash
cd nirimod-ru
git pull
makepkg -si
```

Если репозиторий на диске уже свежий и просто нужно пересобрать без повторного скачивания источников:

```bash
makepkg -si --holdver
```

### install.sh

Повторно запустите установочный скрипт:

```bash
curl -sSL https://raw.githubusercontent.com/y-tretyakov/nirimod-ru/main/install.sh | bash -s -- --install
```

## Удаление

### Пакет, установленный через makepkg

```bash
sudo pacman -Rns nirimod-ru-git
```

`-Rns` уберёт сам пакет, неиспользуемые зависимости и его конфигурационные файлы из системных путей (личные настройки в `~/.config/nirimod/` пакет не трогает).

### install.sh

```bash
curl -sSL https://raw.githubusercontent.com/y-tretyakov/nirimod-ru/main/install.sh | bash -s -- --uninstall
```

## Сборка из исходников (без пакета)

```bash
git clone https://github.com/y-tretyakov/nirimod-ru.git
cd nirimod-ru
uv sync
uv run nirimod
```

Для запуска из исходников системные GTK/PyGObject-зависимости всё равно должны быть установлены через пакетный менеджер дистрибутива.

## Проверка и разработка

Установите dev-зависимости и запустите тесты:

```bash
uv sync --dev
uv run pytest
```

Для проверки PKGBUILD перед коммитом:

```bash
makepkg -f       # пересобрать без переустановки, проверить что сборка вообще проходит
namcap PKGBUILD  # линтер PKGBUILD: ловит нарушения соглашений об именовании, зависимостях и т.д.
namcap *.pkg.tar.zst   # линтер уже собранного пакета: битые зависимости, лишние файлы
```

Если вы обновляете русскую локализацию, старайтесь держать изменения небольшими и совместимыми с оригинальным NiriMod, чтобы будущие обновления из upstream было проще переносить.

## FAQ

### Чем NiriMod RU отличается от оригинального NiriMod?

Это русскоязычный форк: интерфейс, desktop-интеграция, документация и пакетные инструкции адаптированы для русскоязычных пользователей. Основная логика приложения должна оставаться совместимой с оригинальным проектом.

### Можно ли установить NiriMod RU рядом с оригинальным NiriMod?

Нет. Пакет `nirimod-ru-git` конфликтует с `nirimod` и `nirimod-git`, потому что устанавливает тот же исполняемый файл `nirimod`.

### Приложение заменяет мой конфиг?

Нет. Оно редактирует существующую конфигурацию niri, проверяет её через `niri validate` и поддерживает резервные копии. Тем не менее перед активным редактированием сложной конфигурации полезно иметь собственную копию файлов.

### Что делать, если niri не запущен?

NiriMod RU может открыть и сохранить конфигурацию, но применить её через IPC сможет только при запущенном niri. После запуска композитора перезагрузите конфигурацию или нажмите "Сохранить и применить" снова.

### Где хранится конфигурация приложения?

Настройки NiriMod RU и резервные копии по умолчанию лежат в `~/.config/nirimod/`. Конфиг niri по умолчанию читается из `~/.config/niri/config.kdl`.
